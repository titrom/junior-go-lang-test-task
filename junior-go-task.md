# Тестовое задание для Junior Go-разработчика

## Название

HTTP-сервис для учета личных задач

## Цель

Разработать небольшой REST API на Go для управления списком задач. Задание проверяет базовое владение языком Go, работу с HTTP, JSON, структурами данных, ошибками, тестами и организацией проекта.

## Описание

Нужно реализовать сервис, который позволяет создавать, получать, обновлять, завершать и удалять задачи. Данные можно хранить в памяти процесса, без базы данных.

Сервис должен запускаться локально и принимать HTTP-запросы.

## Функциональные требования

### Модель задачи

У задачи должны быть поля:

- `id` - уникальный идентификатор задачи.
- `title` - название задачи.
- `description` - описание задачи.
- `status` - статус задачи.
- `created_at` - дата и время создания.
- `updated_at` - дата и время последнего изменения.

Допустимые значения `status`:

- `new`
- `in_progress`
- `done`

### API

Реализуй следующие эндпоинты:

#### Создание задачи

`POST /tasks`

Тело запроса:

```json
{
  "title": "Купить продукты",
  "description": "Молоко, хлеб, сыр"
}
```

Ожидаемое поведение:

- Если `title` пустой, вернуть ошибку `400 Bad Request`.
- При успешном создании вернуть `201 Created`.
- Новая задача должна получать статус `new`.

#### Получение списка задач

`GET /tasks`

Ожидаемое поведение:

- Вернуть список всех задач.
- Поддержать необязательный query-параметр `status`.

Пример:

`GET /tasks?status=done`

Если `status` передан, вернуть только задачи с указанным статусом.

#### Получение задачи по ID

`GET /tasks/{id}`

Ожидаемое поведение:

- Если задача найдена, вернуть ее.
- Если задачи нет, вернуть `404 Not Found`.

#### Обновление задачи

`PUT /tasks/{id}`

Тело запроса:

```json
{
  "title": "Купить продукты и бытовую химию",
  "description": "Молоко, хлеб, сыр, порошок",
  "status": "in_progress"
}
```

Ожидаемое поведение:

- Можно обновить `title`, `description` и `status`.
- Если `title` пустой, вернуть `400 Bad Request`.
- Если `status` не входит в список допустимых значений, вернуть `400 Bad Request`.
- Если задачи нет, вернуть `404 Not Found`.
- При успешном обновлении вернуть обновленную задачу.

#### Завершение задачи

`PATCH /tasks/{id}/done`

Ожидаемое поведение:

- Изменить статус задачи на `done`.
- Обновить поле `updated_at`.
- Если задачи нет, вернуть `404 Not Found`.

#### Удаление задачи

`DELETE /tasks/{id}`

Ожидаемое поведение:

- Если задача существует, удалить ее и вернуть `204 No Content`.
- Если задачи нет, вернуть `404 Not Found`.

## Нефункциональные требования

- Используй Go `1.22` или новее.
- Можно использовать только стандартную библиотеку Go.
- Данные хранятся в памяти.
- Сервис должен быть безопасен для одновременных запросов.
- Ответы должны быть в формате JSON, кроме `204 No Content`.
- Ошибки должны возвращаться в едином формате.

Пример ошибки:

```json
{
  "error": "title is required"
}
```

## Что нужно покрыть тестами

Минимум:

- Создание задачи.
- Ошибка при создании задачи без `title`.
- Получение задачи по ID.
- Получение `404` для несуществующей задачи.
- Фильтрация списка задач по `status`.
- Обновление статуса задачи.
- Удаление задачи.

Для HTTP-тестов можно использовать `net/http/httptest`.

## Примеры curl-запросов

Создать задачу:

```bash
curl -X POST http://localhost:8080/tasks \
  -H "Content-Type: application/json" \
  -d '{"title":"Read Go docs","description":"Learn net/http package"}'
```

Получить список:

```bash
curl http://localhost:8080/tasks
```

Получить задачи со статусом `new`:

```bash
curl http://localhost:8080/tasks?status=new
```

Завершить задачу:

```bash
curl -X PATCH http://localhost:8080/tasks/1/done
```

Удалить задачу:

```bash
curl -X DELETE http://localhost:8080/tasks/1
```

## Критерии приемки

Задание считается выполненным, если:

- Сервис запускается командой `go run ./cmd/server`.
- Все описанные эндпоинты работают.
- Все ответы и ошибки возвращаются в JSON-формате.
- Некорректные входные данные обрабатываются без panic.
- Одновременные запросы не приводят к гонкам данных.
- Команда `go test ./...` проходит успешно.
- Код отформатирован через `gofmt`.
